Phase 4 (threading) implementation complete 2026-06-04 — all four milestones built, tested, and pushed. The full first-turn threading workflow is in place and exercised end-to-end against fakes/a simulated ring; what remains before the phase can be closed is on-machine validation (a read-only visit + one physics shift). M1 — analysis core + active acquisition: Tikhonov pseudo-inverse + calc_cm_step (orbit diff → corrector step, with first-turn downstream-zeroing), a cached .npz response-matrix artifact + loader (no pySC at runtime — D1), and the active arm/trigger/wait path (wfr:TBT:arm + event-48 trigger). M2 — CMD:STEP_CM: corrector apply with a compare-and-set guard (D5 — refuse if a setpoint drifted) + hardware-amp clamp, the committed HCM/VCM channel catalog (96/72), behind a default-off writer. M3 — CMD:INJECT_ONESHOT: the de-boxed srinjectoneshot (TimInjReq + seqBusy sync + fine delay), a mandatory top-off precondition, and a two-layer gun-fire gate (D3 — inhibit=1 default; inhibit=0 needs explicit opt-in). M4 — loop controller + UI: CMD:THREAD_START/CMD:THREAD_STOP running arm→(optional fire)→read→diff→ calc_cm_step→apply with damping, a divergence guard, and an optional RMS convergence exit (D4), plus a browser threading panel. Every command has identical PV + REST effect (8 commands now in the parity canary). Closeout gate: the remaining on-machine steps are listed in docs/phase-4-controlroom-checklist.md (read-only A1/A2 + active B1–B4).

    • A2 — corrector catalog confirm: dump family2dev('HCM')/('VCM'), diff against the generated catalogs, confirm the SR01-HCM1 device list (counts 96/72 already match).
  2. 🔴 One physics shift + operator sign-off. On the OSPREY:TEST:TXT:* prefix first:
    • B1 fire one shot with inhibit=1 (no charge) → confirm TimInjReq write, seqBusy sync, fine-delay, confirm-signal, and the top-off precondition refusal.
    • B2 apply a small known CMD:STEP_CM on one corrector → confirm readback moves, the compare-and-set refusal, and the limit clamp; restore.
    • B3 confirm arm/trigger/wait actually arms the real TBT BPMs (piggyback on B1).
    • B4 closed-loop CMD:THREAD_START in inhibit=1; then — only with explicit sign-off + promotion to production — real threading with inhibit=0.
    • Bring to the shift: the firing interlock/precondition list (gun permit, shutters, mode) and corrector-limit sign-off (B0 pre-reqs in the checklist).

When A1, A2, and B1–B4 are signed off, Phase 4 is operationally closed.
